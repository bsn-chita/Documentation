# COM соединение.
COM-соединение это более современный и быстрый способ связи между базами 1С по сравнению с OLE. Главное отличие в том, что оно не запускает полноценное приложение с интерфейсом, а создает легкий программный канал к данным.
В терминологии 1С «Внешнее соединение» (External Connection) — это и есть программное название механизма COM-соединения.

Для работы используется объект _V83.COMConnector_.

Ключевые особенности
1. Работает значительно быстрее OLE, так как не инициализирует модули интерфейса и экранные формы.
2. Потребляет меньше ресурсов сервера или клиентского ПК.
3. Ограничения:
    - Нельзя вызвать методы, связанные с интерфейсом (например, Предупреждение(), ОткрытьФорму()).
    - Код в модуле приложения выполняется в специальном режиме (необходимо проверять доступность процедур).
4. Типы данных: Простые типы (числа, строки, даты) передаются напрямую. Ссылочные объекты (Справочники, Документы) остаются "там" — в вашей базе они будут иметь тип COMObject.

Модуль внешнего соединения.
В дереве конфигурации это отдельный модуль, который выполняется только при таком типе подключения. В нем инициализируются параметры сеанса для COM-запросов. Когда вы заходите через «Внешнее соединение», система не выполняет код из «Модуля управляемого приложения» или «Модуля обычного приложения». Работает только «Модуль внешнего соединения».

Права доступа.
В ролях есть отдельное право «Внешнее соединение». Если оно не отмечено, подключиться через V83.COMConnector под этим пользователем не получится.

Директивы.
Если вы пишете код, который должен работать при COM-подключении, его оборачивают в:
```
#Если ВнешнееСоединение Тогда
    // Этот код увидит только COM-коннектор
#КонецЕсли
```

Резюме:
- Минимум ресурсов: Не загружаются графические библиотеки, формы, стили и картинки. Инициализируется только ядро и прикладные объекты.
- Изоляция кода: Выполняется только Модуль внешнего соединения. Код, прописанный в Модуле управляемого приложения (например, привязанный к старту системы), игнорируется.
- Прямой доступ: Вы работаете с объектами базы (документами, справочниками, запросами) напрямую через точку, как если бы писали код внутри самой базы.
- Ограничения: Из-за отсутствия интерфейса попытка вызвать любой диалог или метод Предупреждение() в этом режиме приведет к ошибке.

Основные отличия OLE от COM
- Интерфейс: OLE запускает копию программы с графической оболочкой (её можно сделать видимой через БазаИсточник.Visible = Истина), тогда как COM работает в фоновом режиме без интерфейса.
- Производительность: COM-соединение работает быстрее, так как не тратит ресурсы на отрисовку окон.
- Функциональность: В COM могут быть недоступны некоторые функции, вызывающие диалоговые окна или специфические методы интерфейса.

```
&НаСервере
Процедура COM_СоединениеНаСервере()

	БазаИсточник = Новый COMОбъект("V83.ComConnector"); // COM-объекты поддерживается только в операционных системах Windows
	Попытка
		// Файловая ИБ
		ВнешнееСоединение = БазаИсточник.Connect("File=""/home/user/1C/bases1c/DemoSSl3_1"";USR=""Администратор"";PWD="""";");
	Исключение
	     Сообщить( ОписаниеОшибки());
		 Возврат;
	КонецПопытки;

	Выборка = ВнешнееСоединение.Справочники._ДемоНоменклатура.Выбрать();
	Пока Выборка.Следующий() Цикл
		Сообщить(Выборка.Найменование);
	КонецЦикла;

	Запрос = ВнешнееСоединение.NewObject("Запрос");
	Запрос.Текст = "";
	Результат = Запрос.Выполнить();
	// Выбрать() не работает, но в Выгрузить() работает.
	ТЗ = Результат.Выгрузить();
	Для Каждого Стр Из ТЗ Цикл
		Сообщить(Стр.Наименование);
	КонецЦикла;

КонецПроцедуры

```

> Если вы видите в названии объекта слово "Application", то это OLE объект. OLE Application это полноценное приложение (как если бы его запустил пользователь), а COM это маленький объект для общения с другой ИБ. COM работает намного быстрее.

Внешнее соединение это запуск приложения 1С в специальном режиме. При этом интерфейс не загружается. Это способ перекачки данных, но ресурсов тратиться намного меньше.
