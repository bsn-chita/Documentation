# XML
XML - формат файлов для обмена данными.

## Структура XML
XML состоит из:
- Заголовок или пролог (<?xml ... ?>). это самая первая строка, которую выводит метод ЗаписатьОбъявлениеXML(). Она стоит перед всеми элементами. Не является «элементом» в привычном смысле.
У него нет закрывающего тега в конце файла.
- Элемент (<Тег> ... </Тег>). Это основной контейнер данных. Всегда имеет имя. Может содержать в себе другие элементы, текст или быть пустым.
- Атрибут. Атрибут всегда находится внутри открывающего тега. Он описывает краткие характеристики элемента, которые обычно не содержат переносов строк или сложной структуры.
- Текст. Текст находится между открывающим и закрывающим тегами. Это основные данные, которые несет в себе элемент.

```xml
<ИмяЭлемента ИмяАтрибута1="ЗначениеАтрибута1" ИмяАтрибута2="ЗначениеАтрибута2"...>
    Текст ИмяЭлемента
    <ИмяПодчиненногоЭлемент ...>
    ...
    </ИмяПодчиненногоЭлемент>
</ИмяЭлемента>
```

Ограничения:
- Возможен только один корневой тег

Для программной работы с XML-файлами в языке 1С используются три ключевых механизма:
1. Последовательное чтение и запись (**ЧтениеXML** и **ЗаписьXML**).
Файл обрабатывается элемент за элементом. Это самый быстрый способ, не требующий больших затрат оперативной памяти.
    - Ручной перебор (в цикле)
    Вы сами управляете процессом: «шагаете» по узлам, проверяете имена тегов через Если...Тогда, вручную выцепляете атрибуты и текст.
    - Сериализация
    Вы используете методы ПрочитатьXML() или ЗаписатьXML(). Здесь 1С сама берет объект (например, ДокументОбъект.Реализация) и мгновенно превращает его в XML (или наоборот). Платформа «под капотом» запускает тот же самый последовательный перебор, но делает его сама по жестким внутренним правилам. Вы получаете сразу готовый объект со всеми заполненными реквизитами, табличными частями и движениями.
2. Использование XDTO (XML Data Transfer Objects).
Данные сопоставляются с объектной моделью 1С через специальные XDTO-пакеты (XSD-схемы).
3. Объектная модель документа (DOM).
XML-файл полностью загружается в память в виде дерева объектов (ДокументDOM).

# СПРАВКА ПО КОМАНДАМ
## ЗаписатьОбъявлениеXML()
ЗаписатьОбъявлениеXML() используется для формирования стандартного заголовка XML-файла (пролога). В нем отражается кодировка файла, версия xml.
```xml
<?xml version="1.0" encoding="UTF-8"?>
```
## XMLСтрока()
XMLСтрока() это специальный инструмент в 1С, который переводит данные из типов 1С (числа, даты, булево, ссылки) в строковый формат, понятный для XML. Она не создает теги или атрибуты, а просто готовит значение для записи.
В XML нельзя просто так записать дату или ссылку «как в 1С». Например, дата в 1С выглядит как 01.01.2023 15:00:00, а XML требует стандарт ISO: 2023-01-01T15:00:00. Функция XMLСтрока() делает это преобразование автоматически.

|Тип в 1С    |Результат XMLСтрока(Значение)                               |
|------------|------------------------------------------------------------|
|Число       |100.5 (всегда с точкой, без пробелов-разделителей)          |
|Дата        |2023-10-25T14:30:00 (международный стандарт)                |
|Булево      |true или false (вместо Да/Нет)                              |
|Ссылка      |5e985116-646e-11ee-811c... (Уникальный идентификатор — GUID)|
|Перечисление|Имя значения перечисления (как оно задано в Конфигураторе)  |

Важный момент: Если вы используете СериализаторXDTO или метод ЗаписатьXML(Запись, Объект), то 1С сама вызывает XMLСтрока() под капотом.

## XMLЗначение()
Функция XMLЗначение() из строки (универсального XML типа) преобразует в 1С-ный тип. Она берет «плоскую» строку из XML-файла и превращает её в полноценный тип данных 1С (Число, Дату, Булево или Ссылку).
```
Результат = XMLЗначение(Тип("НужныйТип"), "СтрокаИзXML");
```
Когда вы читаете файл через ЧтениеXML, вы получаете значения как текст. Чтобы не конвертировать их вручную (не резать строку даты, не заменять точку на запятую в числах), используется XMLЗначение.

## УстановитьCтроку()
Указывает что выгрузка будет не в файл, а в строку. XML можно сформировать не в виде файла, а в виде строки. После вызова этого метода все команды записи (ЗаписатьНачалоЭлемента, ЗаписатьАтрибут и т.д.) «накапливают» текст внутри объекта. Итоговая сформированная строка возвращается методом .Закрыть().
Так же может применяться и для работы с ЧтениеXML.

|Действие |В файл (на диск)          |В строку (в память)            |
|---------|--------------------------|-------------------------------|
|Запись   |Запись.ОткрытьФайл("путь")|Запись.УстановитьСтроку()      |
|Чтение   |Чтение.ОткрытьФайл("путь")|Чтение.УстановитьСтроку(Строка)|
|Результат|Создается файл на диске   |Стр = Запись.Закрыть()         |

Важное правило: Для очень больших XML-файлов (сотни мегабайт) всегда лучше использовать диск (ОткрытьФайл), чтобы не вызвать ошибку нехватки оперативной памяти («Недостаточно памяти»).

# XML сериализация

XML сериализация это автоматическое преобразование объектов в памяти в XML.

> _ВозможностьЧтенияXML()_ это функция глобального контекста, которая проверяет может ли текущий фрагмент данных в XML-файле быть автоматически преобразован (десериализован) в объект или значение типа "1С Предприятие". Метод считывает тип данных XML из объекта _ЧтениеXML_ и определяет, существует ли в текущей конфигурации 1С соответствующий ему тип (аналогично методу _ПолучитьXMLТип_). Обычно используется в цикле перед вызовом функции _ПрочитатьXML()_. Если метод возвращает _Истина_, значит данные можно прочитать как объект 1С (например, справочник, документ или массив). Метод работает по принципу **проверки метаданных**, а не по детальному сравнению состава всех реквизитов "поле в поле". 1С анализирует заголовок элемента (смотрит на атрибуты узла XML и ищет там указание на тип данных 1С например, _CatalogObject.Контрагенты_). Система проверяет, существует ли в ИБ объект с таким типом (именем). Метод вернет _Истина_ если имя типа совпало (внутри элемента не проверяются на совпадение реквизиты, их количество или названия). Если _ВозможностьчтенияXML_ вернула _Истина_, но при этом структура внутри XML отличается, то: метод _ПрочитатьXML()_ **пропустит лишние поля (которых нет в ИБ)**, для отсутствующих в XML полей объекта 1С останутся **пустые значения**, ошибка возникнет только в случае, если XML схема в принципе нарушена или тип данных не может быть приведен к типу 1С (например, текст вместо даты).

XML сериализация используется при передаче между клиентом и сервером в управляемых формах и в - _Версионирование объектов_.

Ручной режим
Файл обрабатывается элемент за элементом. Это самый быстрый способ, не требующий больших затрат оперативной памяти.

