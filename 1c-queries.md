# Язык запросов 1С

ВНУТРЕНЕЕ СОЕДИНЕНИЕ
В результат попадают только те строки из обеих таблиц, для которых выполняются условия сопоставления (обычно по равенству ключевых полей).

Синтаксис
```bsl
```

> Если в визуальном конструкторе на вкладке "Связи" убрать галочки "Все" для обеих таблиц, 1С сформирует именно внутреннее соединение.

Внетреннее соединение обычно работает быстрее левого, так как СУБД может эффективно отсекать ненужные данные и использовать индексы.

Замена условия ГДЕ Поле В (ВЫБРАТЬ ...) на внутреннее соединение это классический прием оптимизации в 1С, особенно для PostgreSQL. В PostgreSQL оператор IN (B) с подзапросом часто заставляет оптимизатор выполнять вложенный запрос для каждой строки основной таблицы.


ВЫРАЗИТЬ - защита от неявных соединении

Параметры виртуальных таблиц (отборы делаются в параметрах, а не в секции ГДЕ)
