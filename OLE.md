# OLE соединение.
OLE-соединение (Object Linking and Embedding) в 1С позволяет управлять одной информационной базой из другой или из внешнего приложения. Сейчас чаще используется более быстрый механизм COM-соединения, но OLE остается актуальным, если нужно запустить полноценное приложение с интерфейсом.

> OLE-соединение можно использовать для перекачки данных между конфигурациями (МЕДЛЕННО).

## Создание объекта.
В зависимости от версии 1С используются разные идентификаторы.

```bsl
// Например, V83.Application запускает «толстый клиент» 1С:Предприятие 8.3.
Новый COMОбъект("V83.Application")
```

## Строка подключения.
Параметры зависят от типа базы (файловый или серверный вариант). В окне "Запуск 1С Предприятия" в нижней части отображается путь к информационной базе (строка подключения). Копируем ее и используем для параметров при настройке COM или OLE соединений.
Для серверного варианта будет строка вида:
```bsl
Srvr="ИмяСервера";Ref="ИмяБазы";
```
Для файлового варианте будет строка вида:
```bsl
File="c:\Path\To\Base";
```
Для успешного подключения одного пути не достаточно. В строку подключения обязательно нужно добавить параметры авторизации, даже если они пустые.

Вот так должна выглядеть полная строка для:

1. Для файловой базы:
```bsl
File="ПутьКБазе"; Usr="Логин"; Pwd="Пароль";
```
Поскольку в коде это будет строка в которой присутствуют кавычки их нужно экранировать.
```bsl
БазаИсточник.Connect("File=""c:\Path\To\Base"";USR=""Администратор"";PWD="""";");
```

2. Для серверной базы:
```bsl
Srvr="ИмяСервера"; Ref="ИмяБазы"; Usr="ИмяПользователя"; Pwd="Пароль";
```

```bsl
БазаИсточник.Connect("Srvr=""ИмяСервера""; Ref=""ИмяБазы""; Usr=""ИмяПользователя""; Pwd=""Пароль"";");
```
